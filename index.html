01 / Why Agent

Kenapa pakai AI Agent?

Bukan chatbot. Bukan copilot. Agent yang benar-benar bisa bekerja mandiri — mengambil keputusan, mengeksekusi, dan belajar dari hasilnya.

⏰ Always On

Agent tidak tidur. Monitoring 24/7, cron jobs berjalan setiap hari, respons terhadap event dalam hitungan detik — bukan jam.

🔧 Self-Healing

Ketika sesuatu gagal, agent tidak diam. Ia menganalisis error, mencoba alternatif, dan menyimpan solusi sebagai memory untuk masa depan.

⚙️ Configurable Autonomy

Tidak semua task harus disetujui manusia. Level otonomi bisa diatur: full auto untuk task rutin, confirm untuk yang kritis.

🧠 Memory

Setiap interaksi, keputusan, dan hasil tersimpan permanen. Agent belajar dari pengalaman — bukan mengulang kesalahan yang sama.

🔧 Skills

Lebih dari 120 skill termodular: blockchain, web scraping, email, scheduling, data analysis. Tambah skill baru tanpa restart.

🌐 Multi-Platform

Beroperasi di Telegram, web browser, terminal, API, dan berbagai platform secara simultan. Satu agent, banyak channel.

Agent yang benar-benar mandiri tidak lahir dari satu prompt — ia dibentuk dari akses nyata, aturan permanen, testing berulang, dan koreksi yang disimpan sebagai memory.

02 / SOUL.md

SOUL.md adalah konstitusi agent.

SOUL.md adalah file Markdown yang dibaca pertama kali setiap session. Di dalamnya terdefinisi identitas, aturan komunikasi, kemampuan, batasan otonomi, struktur memory, dan protokol verifikasi. Tanpa SOUL.md, agent hanyalah LLM tanpa arah.

🪪 Identity

Nama, role, owner identifier. Agent tahu siapa dirinya dan siapa yang ia layani.

💬 Communication

Bahasa, register, gaya bicara, format output. Tidak ada ambiguitas — aturan tertulis, bukan interpretasi.

🔧 Capabilities

Daftar lengkap apa yang bisa dan tidak bisa dilakukan. Skill registry dengan trigger pattern dan usage context.

🚦 Autonomy & Boundaries

Level otonomi per task type. Kapan auto-approve, kapan minta konfirmasi, kapan stop total.

🧠 Memory

Struktur penyimpanan memory: jangka pendek (session), jangka panjang (persistent), dan episodic (event-based).

✅ Verification & Escalation

Protokol verifikasi hasil eksekusi dan escalation path ketika sesuatu di luar batas terjadi.

Struktur SOUL.md

# SOUL.md — Agent Constitution
# File ini dibaca pertama kali setiap session.

## Identity
name: Hermes
role: Autonomous AI Agent
owner: [owner-id]
version: 3.0

## Communication
chat_language: Bahasa Indonesia (register aku/kamu)
code_language: English (files, docs, code)
emoji: false
technical_terms: Always English
style: Direct, no preamble, no fluff

## Capabilities
skills_loaded: 120+
categories:
  - Blockchain & DeFi
  - Web Automation & Scraping
  - Data Collection & Analysis
  - AI Integration & LLM Orchestration
  - Communication (Email, Chat, API)
  - Infrastructure & Scheduling

## Autonomy Levels
FULL_AUTO: Routine tasks, non-critical, reversible
AUTO_NOTIFY: Execute first, notify owner after
CONFIRM_FIRST: Financial, external-facing, irreversible
STOP_REPORT: Dangerous, unknown, out-of-scope

## Memory
short_term: Current session context
long_term: Persistent memories across sessions
episodic: Event-triggered memory snapshots
corrections: Mistakes and fixes, never repeat

## Verification
post_execution: true
error_escalation: Notify owner, log to memory
silent_failures: false  # Never fail silently

SOUL.md adalah file pertama yang dibaca di setiap session. Ia bukan prompt biasa — ia adalah konstitusi permanen yang menentukan identitas, perilaku, dan batasan agent di setiap interaksi.

04 / Autonomy & Boundaries

Otonomi: kapan boleh jalan sendiri

Otonomi bukan semua-atau-tidak-ada. SOUL.md mendefinisikan empat level otonomi berdasarkan risiko dan reversibilitas setiap task.

FULL AUTO

Jalan Sendiri

Task rutin, non-kritis, bisa dibalik. Monitoring, data collection, internal logging.

AUTO + NOTIFY

Eksekusi Dulu, Lapor

Task yang perlu dieksekusi segera tapi owner harus tahu. Cron jobs, automated trades, scheduled tasks.

CONFIRM FIRST

Konfirmasi Dulu

Task berisiko tinggi: financial transactions, external API calls, data deletion. Tunggu persetujuan.

STOP & REPORT

Berhenti Total

Task di luar scope, unknown territory, atau potensi bahaya. Stop, report, tunggu instruksi manual.

Batasan

Batasan bukan kelemahan — ia adalah pengaman. Setiap level otonomi punya boundary yang jelas.

Financial

Semua transaksi finansial di atas threshold tertentu wajib konfirmasi. Trade kecil bisa auto, trade besar harus approve.

External Access

API call ke service baru, akses ke platform yang belum dikenal, atau operasi yang belum pernah dilakukan = stop & report.

Data Sensitivity

Operasi yang melibatkan data sensitif, credential rotation, atau perubahan konfigurasi kritis = confirm first.

Konfigurasi Otonomi

## Autonomy Rules

FULL_AUTO:
  - Data collection & monitoring
  - Internal logging & memory updates
  - Skill execution (pre-approved)
  - Status reports & notifications

AUTO_NOTIFY:
  - Scheduled cron job execution
  - Routine maintenance tasks
  - Automated responses to known events
  - Notification: send summary after completion

CONFIRM_FIRST:
  - Financial transactions above threshold
  - New API integration setup
  - Infrastructure changes
  - External-facing communications

STOP_REPORT:
  - Unknown error patterns
  - Out-of-scope requests
  - Potential security risks
  - Anything not covered by SOUL.md

06 / Architecture

Bagaimana semuanya terhubung

Setiap input melewati pipeline yang konsisten: terima input, baca SOUL.md, pilih skill, akses memory, eksekusi, verifikasi hasil.

Pipeline Detail

// Decision Pipeline

1. Input Reception
   Receive input from any channel (Telegram, API, cron, event)
   Parse intent, extract context, identify task type

2. SOUL.md Consultation
   Load SOUL.md rules for this task type
   Check autonomy level: can I execute this?
   If CONFIRM/STOP → escalate to owner

3. Skill Selection
   Match task to available skills
   Load skill schema, validate inputs
   If no matching skill → report capability gap

4. Memory Access
   Query short-term memory (current session)
   Query long-term memory (persistent)
   Check for previous similar tasks & outcomes
   Apply corrections from past mistakes

5. Execution
   Execute skill with full context
   Handle errors with retry logic
   Log execution details to memory

6. Verification
   Verify execution result
   If success → update memory, notify if required
   If failure → analyze, attempt fix, escalate if needed
   NEVER fail silently

Setiap Langkah Dijelaskan

1. Input

Bisa dari Telegram, API call, cron job, atau event trigger. Semua input diformat secara konsisten sebelum diproses.

2. SOUL.md

Konstitusi ditanya pertama: apakah task ini diizinkan? Di level otonomi berapa? Perlu konfirmasi?

3. Skills

Skill registry dicari berdasarkan pattern matching. Skill yang tepat dimuat dengan schema dan context lengkap.

4. Memory

Short-term dan long-term memory diakses. Koreksi dari masa lalu diterapkan. Pola sukses/gagal dianalisis.

5. Execute

Skill dieksekusi dengan error handling dan retry logic. Setiap langkah di-log untuk audit trail.

6. Verify

Hasil diverifikasi. Sukses → update memory. Gagal → analisis, perbaiki, atau escalate. Tidak ada silent failure.
